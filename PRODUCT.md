# 321Doit 产品事实源

> 本文件是 321Doit 对外介绍、网站、README、发布文案与 AI Agent 生成内容的首要事实来源。若其他旧文档与本文件冲突，以本文件和当前应用内的 `ToolRegistry` 为准。

## 一句话定位

**321Doit 是一款免费、开源、本地优先的 macOS 影视制作全能工作站。**

它覆盖从前期构思、分镜设计、拍摄统筹、现场场记，到素材安全下盘、媒体检查与格式转换，再到后期交接的一体化生产流程。它已经不是早期的单一拷卡工具，也不应再以“321开拷”作为产品主名称或核心定位。

英文标准表述：

> **321Doit is a free, open-source, local-first filmmaking workstation for macOS.**

## 当前五个核心工具

1. **灵动分镜（Living Storyboard）**
   - 结构化分镜表与故事板
   - 画面分层画布、图片元素、绘画与快捷键
   - 景别、角度、运镜导演轮
   - 人物走位、摄影机路径、机位与等效焦段视场
   - 动态分镜时间线、预演、检查与导出

2. **拍摄统筹（Production Planning）**
   - 拍摄日历、场景计划与每日通告
   - 人员、地点、时间安排和现场信息
   - 与分镜、场记及项目工作区衔接

3. **迅捷场记（Rapid Script Log）**
   - 场、镜、次、多机位 Take 与连戏信息
   - 键盘优先操作和 iPad 伴随版本
   - 可导入剪辑软件并承接分镜与拍摄计划数据

4. **极速拷卡（Turbo Offload）**
   - 最多三个目标的安全下盘
   - xxHash64、MD5、SHA-1、SHA-256 校验
   - 回读验证、严格续传、预检、ASC MHL 与多格式报告
   - 代理、LUT、项目任务和后期桥接能力

5. **媒体转换（Media Conversion）**
   - MP4、MOV、MKV、WebM、AVI、M2TS、MXF 等容器
   - H.264、H.265、ProRes、AV1、VP9、MPEG-2、DNxHR 等视频编码
   - 仅换封装、视频转码与无损音频转换
   - 读取时长、分辨率、帧率、码率、像素格式、音频和色彩元数据
   - 转换前兼容性检查、过程进度、结果验证与可追溯报告

## 产品结构

321Doit 采用“工具工作站 + 项目上下文”的结构：

- 默认使用**项目工作流**：点击五个工具时先选择或新建项目，再进入对应工具。
- 首页勾选“**不使用项目 · 独立使用工具**”后，五个工具可不关联项目直接打开。
- 首页提供“**打开项目**”和“**AI 模式**”两个独立按钮；独立模式下“打开项目”显示为灰色，点击后提示先关闭独立模式。
- **Mira AI** 是仍在 Beta 验证中的原生协作入口，基于现有受控本地 MCP 接口；不得将其描述为已经完整交付、可无人监管执行的生成式 AI 能力。
- Mira 不固定绑定某一项目；它可在用户明确授权的文件夹或磁盘中发现、新建、读取和修改项目，并可在明确确认后将项目移到 macOS 废纸篓。
- 首次进入 Mira 且尚未配置全盘根目录时，应用会引导用户在 macOS“隐私与安全性 → 完全磁盘访问权限”中添加并启用 321Doit；用户确认进入系统设置后，根目录 `/` 会作为内置 MCP 的允许目录。macOS 要求该系统权限由用户亲自开启，应用不能代替确认。若暂不授权，用户仍可在 Mira 侧栏逐个授权文件夹或磁盘。
- Mira 通过本地 MCP 执行受控操作，并在 321Doit 界面中展示会话、工具状态、错误和写入确认；已打开的同一项目会在 AI 提交变更后即时重载。OpenCode 只作为后台，不向普通用户暴露终端或原始网页。
- Mira 提供“全自动”和“妈宝模式（每次写入前确认）”两种执行权限；默认使用后者。两种模式都只作用于 321Doit 的受控工具与用户明确授权的位置，不开放任意终端或任意文件访问。
- Mira 不提供、共享、代管或默认启用任何模型额度。首次使用时，用户必须自行配置 OpenAI-compatible API、自己的 OpenCode Go 订阅 Key，或本地模型服务；凭据仅保存在本机 macOS Keychain。项目内容仅在任务需要时按需读取并随用户选择的模型服务请求发送，费用、可用额度、数据政策与服务条款由该服务商直接管理。
- 不宣传或默认接入第三方“免费模型”。即使某服务商临时提供免费额度，也必须由用户以自己的账户和 Key 主动选择，并遵守其数据政策、试用限制与服务条款；不得共享、转售、代管或暗示 321Doit 提供该额度。
- 各工具是同一工作站中的专业模块，不是彼此无关的小应用。
- 当前已交付 Beta 阶段的本地 MCP 工具接口：五个模块均可向兼容 AI 客户端提供结构化读取和实际工作入口。
- Agent 可受控地编写完整分镜场次、修改分镜方案、填写拍摄日与通告、记录场记 Take、执行校验拷卡和媒体转换，并取得进度、产物路径与报告路径。
- 本地 MCP 接口不等于内置生成式 AI。它不调用云端模型，必须显式授权目录；写入或执行必须经过客户端确认、幂等键及各模块原有的校验、备份、字段锁、修订与审计约束，不应宣传成无人监管的自动执行。

## 标准产品叙事

推荐使用以下顺序介绍产品：

```text
创意与剧本意图
  → 灵动分镜
  → 拍摄统筹
  → 迅捷场记
  → 极速拷卡
  → 媒体转换
  → 剪辑与后期交接
```

安全拷卡仍然是 321Doit 的重要专业能力和可信基础，但不再代表整个产品。

## 品牌与文案约束

- 主名称只使用 **321Doit**。
- 中文定位使用“**影视制作全能工作站**”或“**影视全流程工作站**”。
- 不再把整个产品称为“DIT Copy Station”“拷卡站”或“321开拷”。
- 可以在项目历史中说明它“最初从安全拷卡起步”，但不能写成“正在成为工作站”或“完整工作站仍是长期路线图”。
- 面向网站和发布内容时，应优先展示五个当前工具，而不是把绝大多数篇幅用于拷卡。
- 不虚构云协作、订阅服务或超出已验证本地 MCP 接口的 AI 功能。321Doit 的当前原则是免费、开源、本地优先。

## 目标用户

- 独立电影、纪录片、广告、MV、短片与学生剧组
- 导演、摄影、制片、场记、DIT、剪辑与后期人员
- 需要一套本地、透明、可验证且不依赖高额订阅的影视工作流的小型团队与个人创作者

## 当前状态

321Doit 仍处于 Beta 阶段，但“影视制作全能工作站”已经是当前产品形态，而不是未来愿景。各模块成熟度不同，具体功能状态应以应用内实际界面、自动化测试和发布说明为准。
