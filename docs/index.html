五个工具，一条链路Five tools, one chain

不只管理素材，也管理镜头如何诞生。Manage not only media, but how every shot comes to life.

每个工具都能独立使用；进入项目模式后，分镜、拍摄日、场记、素材和交接记录会围绕同一个项目继续流动。Every tool works independently. In Project Mode, storyboards, shooting days, logs, media, and handoff records move together.

01

灵动分镜Living Storyboard

在结构化分镜表里设计画面、景别、角度、运镜、机位、人物走位与动态预演。Design frames, shot sizes, angles, movement, camera blocking, talent paths, and animatics in a structured shot table.

02

拍摄统筹Production Planning

把场景拆进拍摄日历与通告，安排时间、地点、人员和现场信息。Schedule scenes into shooting days and call sheets with time, locations, crew, and on-set details.

03

迅捷场记Rapid Script Log

快速记录场、镜、次、多机位、圈选、失误与连戏，让计划变成真实拍摄记录。Capture scenes, shots, takes, multicam, circle takes, errors, and continuity as production happens.

04

极速拷卡Turbo Offload

最多三目标安全下盘，执行预检、回读校验、严格续传并生成 ASC MHL 与报告。Offload to up to three destinations with preflight, read-back verification, strict resume, ASC MHL, and reports.

05

媒体转换Media Conversion

读取媒体参数，完成换封装、视频转码与无损音频转换，并验证输出结果。Inspect media, rewrap streams, transcode video, convert lossless audio, and verify every output.

06

后期交接Post Handoff

把镜头意图、现场记录、素材、校验凭证和技术元数据一起交给剪辑与后期。Hand shot intent, set records, media, verification proof, and technical metadata to editorial and post together.

极速拷卡 · 安全不是提示框Turbo Offload · Safety is not a dialog

在写入之前，先替你想到最坏情况。Think through the worst case before writing begins.

片场最可怕的不是报错，而是拷完了才发现一开始就不该这么拷。321Doit 的预检、校验和审计结构，都是为了让“素材安全”可被证明。The scary part is not an error; it is discovering after copy that the job should never have started. Preflight, verification, and audit structure make media safety provable.

  • 源卡可读 / 目标盘可写 / 容量足够Source readable, targets writable, enough capacity
  • 覆盖风险 / 同名卡冲突 / 系统盘警告Overwrite risk, duplicate card names, system drive warnings
  • 同物理盘 / 文件系统限制 / 隐藏异常文件Same physical volume, filesystem limits, hidden abnormal files
  • 可恢复旧任务识别 / 严格续传验证Resumable session detection and strict resume validation

FAQ

小团队关心的问题Questions small crews ask

没有 FFmpeg 可以用吗？Can I use it without FFmpeg?

可以。核心拷贝、校验、基础报告和基础 MHL 不依赖 FFmpeg；高级代理、LUT、截图和专业格式处理建议使用 FFmpeg。正式离线安装包已包含对应组件。Yes. Core copy, verification, basic reports, and basic MHL work without FFmpeg. Advanced proxies, LUT, frame grabs, and pro formats benefit from FFmpeg; the offline installer includes it.

为什么第一次打开需要右键 Open？Why right-click Open on first launch?

当前版本主要通过 GitHub Releases 分发，部分构建可能是 ad-hoc 签名而不是 Apple Developer ID notarized。首次通过 Gatekeeper 后即可正常双击打开。Releases are distributed via GitHub, and some builds may be ad-hoc signed rather than Apple Developer ID notarized. After first Gatekeeper approval, normal double-click launch works.

它会不会把素材上传到云端？Does it upload media to the cloud?

不会。321Doit 的理念是本地优先：素材、校验、报告和项目数据都围绕本地项目工作区展开，不强制上云，也不制造订阅锁定。No. 321Doit is local-first: media, checksums, reports, and project data stay around your local workspace. No forced cloud lock-in or subscription.

适合什么团队？Who is it for?

独立电影、纪录片、学生剧组、小型广告、MV、地方制作团队、单人创作者，以及所有希望用得起、看得懂、信得过专业流程的人。Independent filmmakers, documentaries, student crews, small commercial teams, MV crews, regional production teams, solo creators, and anyone who wants professional workflow without studio infrastructure.
