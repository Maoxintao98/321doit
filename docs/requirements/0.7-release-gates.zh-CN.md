# 321Doit 0.7 发布门槛

## 版本定位

0.7 是面向 1.0 的可靠性与工作流收敛版本。产品定位保持为“免费、开源、本地优先的 macOS 影视制作全能工作站”，覆盖灵动分镜、拍摄统筹、迅捷场记、极速拷卡和媒体转换五个核心工具。

本版本不把 AI 接口描述成已经交付的能力，也不引入云协作或订阅服务承诺。0.7 正式发布前，官网和公开更新源继续指向当前稳定版 0.6。

## 验证阶段

- **开发基线。** 版本切换到 0.7，锁定版本一致性，补齐报告失败和项目降级迁移回归。
- **故障恢复阶段。** 集中验证断电、拔盘、空间不足、目标盘只读、同名任务、续传和取消。
- **现场验证阶段。** 使用真实相机卡、不同文件系统和 Intel / Apple Silicon Mac 完成矩阵测试。
- **RC：发布候选。** 只接受阻断发布的问题修复，不再增加功能面。

内部 Build 是每次成功构建自动递增的技术编号，不再用 Build 2、Build 3 表示验证阶段。正式候选需要复现时用 `APP_BUILD_OVERRIDE` 锁定编号。

## 自动化硬门槛

- `./run_tests.sh` 全部通过。
- `./run_rigorous_tests.sh` 全部通过，并实际运行固定版本 `ascmhl 1.2` 官方交叉验证。
- Resolve 交接脚本测试全部通过。
- 所有 Swift 文件完成编译或语法检查；Python、JavaScript、Shell、JSON、XML 和 plist 检查通过。
- `./build.sh` 产出 `arm64 + x86_64` Universal 2 应用，严格签名校验通过。
- `build.sh`、命令行回归测试回退值和构建产物 Info.plist 的版本、Build 必须一致。

## 数据与安全门槛

- 拒绝源目录符号链接和逃逸路径，不读取用户未选择的源目录外文件。
- 强制校验报告写入失败不能显示任务成功。
- 项目保存使用原子主快照；旧版本同时写出的较新兼容文件可以安全迁移，单文件撕裂写入不能覆盖主快照。
- CSV 导出必须阻断表格公式注入。
- 更新包只允许经过签名与哈希验证的同一份本地缓存进入安装流程；同一短版本号下可按更高 Build 自动升级。

## 人工验收门槛

- 五个核心工具分别完成新建、保存、关闭、重新打开和导出主流程。
- 中文和英文界面无错位、截断或产品定位不一致。
- 0.6 项目升级到 0.7 后数据完整；必要时用 0.6 再次打开兼容文件，不丢失后续编辑。
- APFS、HFS+、exFAT 目标盘完成真实拷卡；覆盖拔盘、休眠、取消、续传和剩余空间不足。
- Intel 与 Apple Silicon 至少各完成一次冷启动、五工具冒烟和更新检查。
- 发布说明准确描述已交付能力，并明确 Beta 风险；官网、README、应用内版本和下载包一致后才切换公开版本。
