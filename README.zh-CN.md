# 321Doit 

**321Doit** 是一款免费、开源、本地优先的原生 macOS **人与 AI 协作的全能视频工作站**，面向视频创作者、导演、摄影、制片、场记、DIT、剪辑和后期人员，也面向独立创作者与小型团队。

它把视频创意、分镜、拍摄统筹、现场场记、安全拷卡、媒体转换与后期交接放进同一个项目工作区。人可以直接使用五个专业工具，也可以通过用户授权、受控的本地工具与 Mira AI 协作。321Doit 最初从 DIT 安全拷卡起步，但现在已经不是单一拷卡工具。

321Doit 的核心想法很简单：

> 专业的视频工作流不应该只属于预算充足的大团队；AI 协作也应当受控、透明。

## 321Doit 是什么？

**321Doit** 是一套由五个专业工具和 Mira AI 受控协作层组成的 macOS 原生全能视频工作站。每个工具可以独立使用，也可以在项目模式下共享场次、镜头、拍摄日、素材与交接数据：

```text
创意与剧本意图
  → 灵动分镜
  → 拍摄统筹
  → 迅捷场记
  → 极速拷卡
  → 媒体转换
  → Final Cut Pro / DaVinci Resolve 与后期交接
```

它不是 Finder 的替代品。  
不是简单的“复制文件工具”。  
不是只负责生成校验报告的拷卡证明器。  

321Doit 已经在做的是：

> 把创意、镜头、计划、现场记录、素材安全与后期处理连接成一套真正可执行、可追踪的人与 AI 协作视频工作流。

五个当前核心工具的完整事实说明见 [PRODUCT.md](PRODUCT.md)。

## 核心功能

### 本地 AI 工具接口（Beta）

Universal 2 应用内已包含本地 stdio MCP 服务，可供 Codex、OpenCode
等兼容 AI 客户端使用。当前 22 个工具覆盖结构化项目发现、快照读取、
分镜创作、拍摄统筹、现场场记、校验拷卡与媒体转换。

Agent 可以编写完整分镜场次、填写并导出拍摄日通告、记录并导出场记 Take、
执行校验拷卡和媒体转换。长任务会返回实时任务状态，完成后给出明确的产物
与报告绝对路径。写入与执行受到显式授权目录、人工确认、幂等键、项目权限、
字段锁、修订、本地备份、结果验证与审计记录共同约束。

这是本地工具接口，不是内置云端模型：它不会主动联网，AI 客户端目前也不能
删除或格式化源卡，也不能发布未经验证的转换结果。接入方式和客户端配置见
[Tools/MCP/README.md](Tools/MCP/README.md)。

### 安全下盘

321Doit 的核心不是“复制文件”，而是建立一条可信的片场素材交接链路。

它可以将一张摄影机卡或一个素材目录同时复制到多个目标位置，例如：

- 工作盘
- 备份盘
- 归档盘

相比直接使用 Finder 拖拽复制，321Doit 会围绕下盘流程生成校验记录、报告文件、MHL、checksum sidecar 和后期交接结构。

这让素材下盘不只是“文件到了硬盘里”，而是可以被检查、被追踪、被交接。

---

### 多目标复制

321Doit 支持从一个来源写入最多三个目标位置。

这适合小型剧组常见的三盘流程：

```text
摄影机卡
  ↓
工作盘 + 备份盘 + 归档盘
```

如果某一个目标盘出现问题，其他目标盘可以根据具体错误继续执行，避免一个硬盘、接口、线材或连接问题直接拖垮整个下盘任务。

---

### 多种校验算法

321Doit 支持多种校验算法：

- xxHash64
- MD5
- SHA-1
- SHA-256

不同项目可以根据自己的需求选择速度优先或兼容性优先的校验方式。

被选中的校验算法会尽量贯穿拷贝验证、报告生成和交接记录，而不是只在某一个小环节生效。

---

### 断点续传与严格续传

321Doit 支持通过 `session.json` 记录中断任务，并在之后恢复。

同时，321Doit 支持更安全的“严格续传”。

普通续传很容易只看：

- 文件名
- 文件大小
- 修改时间

但这些信息看起来正确，不代表文件内容真的正确。

严格续传模式会重新读取目标文件并计算校验值，确认内容一致后才跳过该文件。

这可以帮助发现：

- 上一次拷贝中断造成的半成品文件
- 文件大小一样但内容被改过的文件
- 硬盘或传输异常导致的隐性错误
- 人为替换但外观看起来相似的文件

---

### 拷贝前安全预检

在正式下盘前，321Doit 会进行预检。

当前预检方向包括：

- 源卡是否可读
- 目标盘是否可写
- 目标盘容量是否足够
- 是否存在覆盖风险
- 是否存在同名卡号冲突
- 是否正在写入系统盘
- 源和目标是否在同一物理磁盘
- 文件系统是否存在限制
- 是否存在隐藏文件或异常文件
- 是否存在空文件夹
- 是否存在可恢复的旧任务

这个模块的意义很直接：

> 片场最可怕的不是报错，而是拷完了才发现一开始就不该这么拷。

### 摄影机卡自动识别

321Doit 可以扫描已挂载的磁盘卷，并推断它可能来自什么设备或流程。

当前识别逻辑覆盖或辅助识别：

- Sony
- Canon
- RED
- ARRI
- Blackmagic Design
- GoPro
- Panasonic
- 微单 / 混合相机
- 小型剧组常见非标准目录结构

这对真实片场很重要。

因为很多小组并不会严格按照工业流程命名卡号、卷名和文件夹。  
321Doit 的目标不是假装片场永远规范，而是在不规范的现实里尽量帮用户兜底。

### 卡号登记与摄影机登记

321Doit 已经加入卡号登记与摄影机登记流程。

这些模块用于帮助剧组追踪：

- 摄影机机身

- 卡号
- 卡使用记录
- 摄影机与卡的对应关系

在小型剧组里，卡号、机身、卷名和素材目录经常靠人工记录。  
321Doit 希望把这些容易出错的信息纳入同一个项目流程中，减少靠记忆、聊天记录和纸面标签维持秩序的风险。

### 代理、转码与 LUT

321Doit 已经不只是拷卡。

它可以进入后期准备流程，生成代理文件、转码文件和 LUT 烧录版本。

当前支持方向包括：

- H.264
- H.265
- ProRes
- H.266 实验流程
- LUT 烧录
- 同时生成带 LUT / 不带 LUT 的代理版本
- 后期交接包

在 macOS 上，321Doit 会优先使用系统能力完成合适的任务，例如 AVFoundation 和 VideoToolbox。  
更复杂的编码、滤镜、LUT、截图和专业格式处理则依赖 FFmpeg。

核心拷卡功能不依赖 FFmpeg。  
但如果需要完整的代理、转码、LUT、截图和专业格式支持，建议安装 FFmpeg。

## 典型工作流

一个当前版本的 321Doit 工作流可能是：

```text
1. 创建或打开项目
2. 登记或确认摄影机与卡号信息
3. 插入摄影机卡
4. 识别卡信息
5. 选择 1 到 3 个目标盘
6. 执行拷贝前预检
7. 开始安全下盘
8. 逐文件校验
9. 生成 MHL / PDF / CSV / JSON / TXT 报告
10. 根据需要生成代理
11. 根据需要烧录 LUT
12. 记录或整理场 / 镜 / 次元数据
13. 根据需要把已验证素材与场记元数据同步到剪辑软件插件
14. 将原始素材、代理、报告、卡号记录和元数据一起交给后期
```

## 安装使用

本软件当前主要通过 GitHub Releases 分发，部分版本可能是 ad-hoc 签名，而不是 Apple Developer ID notarized 版本。

正式离线安装包的使用方式：

1. 打开 DMG。
2. 双击 **安装 321Doit.pkg**，完成 macOS 安装流程。
3. 从 **Applications / 应用程序** 启动 `321Doit.app`。

当前发布通道继续使用 ad-hoc 签名，没有 Apple 公证。安装器会清除它实际安装的 App 的隔离属性，但浏览器手动下载的 PKG 仍可能被 macOS 拦截。请先核对发布页文件名和 SHA-256，再右键 PKG 选择 **打开**；若系统仍阻止，可到 **系统设置 → 隐私与安全性 → 仍要打开**。

只有在 App 绕过正式安装器被单独复制、且需要诊断时，才使用下面的兜底命令：

```sh
xattr -dr com.apple.quarantine /Applications/321Doit.app
```

## FFmpeg 可选依赖

321Doit 的核心拷贝与校验功能不需要 FFmpeg。

没有 FFmpeg 时，仍然可以完成：

- 素材复制
- 文件校验
- 基础报告
- 基础 MHL 输出

建议安装 FFmpeg 以启用或增强：

- 高级代理生成
- H.265 / H.266 工作流
- LUT 烧录
- 画面截图
- 专业编码格式
- 部分 RAW / 专业视频格式解码
- 更完整的转码流程

正式离线安装包已经包含 Universal 2 版 FFmpeg 与 FFprobe。安装器会优先保留电脑上与当前机器原生架构兼容的版本；否则直接使用 321Doit 内嵌组件，不重复复制，不安装 Homebrew，也不联网下载。

开发期间只更新本机 App，不需要反复制作 DMG：

```sh
./update_app.sh
```

只有制作正式发行安装包时才运行 `./package.sh`。321Doit 仍会优先自动检测电脑已有的 FFmpeg，也支持在设置中手动指定路径。

## 从源码构建

需要：

- macOS 13 或更新版本
- Xcode 或 Xcode Command Line Tools
- Swift
- 正式安装包已随附 FFmpeg/FFprobe；从源码开发时可选

克隆仓库：

```sh
git clone https://github.com/Maoxintao98/321doit.git
cd 321doit
```

构建：

```sh
./build.sh
```

普通内部构建会自动取得严格递增的 Build 号。上一次成功的计数保存在被 Git 忽略的 `build/` 目录中；编译失败不会消耗 Build。

需要复现某个正式产物时，可以显式锁定：

```sh
APP_BUILD_OVERRIDE=42 ./build.sh
```

`./update_app.sh` 使用相同的自动计数，并会拒绝用同一版本下相同或更低的 Build 覆盖已安装 App；回滚测试必须显式放行。

构建脚本会尽量生成同时支持 Apple Silicon 和 Intel Mac 的 Universal Binary。

## 自动更新

321Doit 内置 Sparkle 兼容的更新检查机制。

应用能识别同一短版本号下更高的 Build，校验 appcast 中的 Ed25519 安装包签名，把通过校验的同一份 DMG 保存在本机缓存，只对该已验证文件清除隔离属性，然后打开它安装。因此下载、校验和安装使用的是同一个包。appcast 会保留旧版本条目，作为回滚下载入口。

## 当前限制

321Doit 仍处于 Beta 阶段。

已知限制包括：

- 当前主要面向 macOS。
- 当前不通过 Mac App Store 分发。
- 部分版本可能是 ad-hoc 签名，而不是 Apple Developer ID notarized 版本。
- 第一次打开时可能需要手动通过 macOS Gatekeeper。
- 高级代理、LUT、截图和专业编码能力取决于本机 FFmpeg。
- R3D、BRAW、ARRIRAW、CRM 等格式取决于本机 FFmpeg 构建和解码支持。
- iPhone 与 iPad 伴随工作流仍在规划中。
- 从前期到最终交付的完整流程覆盖仍属于长期路线图。
- 正式项目使用前，请先用测试素材完整跑通你的工作流。

## 为什么要做 321Doit？

很多小型剧组、独立创作者、学生团队、纪录片团队，还在用 Finder、手动文件夹、Excel、纸质场记单、截图、聊天记录和记忆管理片场素材。

这不是因为他们不需要专业流程。

很多时候只是因为专业工具太贵、太封闭、太复杂，或者根本不是为小团队设计的。

321Doit 想做的是：

> 让小团队也能拥有可靠、透明、可验证的人与 AI 协作视频工作流。

它不是为了替代大型工业系统。  
它是为了让预算有限、人员有限、但同样重视素材安全和生产秩序的创作者，也能有一个用得起、看得懂、信得过的工具。

它适合：

- 独立电影创作者
- 纪录片团队
- 学生剧组
- 小型广告团队
- MV 团队
- 地方视频制作团队
- 单人创作者
- 所有曾经拷摄影机卡时默默祈祷“千万别出事”的人

## 项目理念

321Doit 是开源的，因为创作工具不应该只属于预算充足的大团队。

一个给创作者使用的生产工具，应该是：

- 安全的
- 透明的
- 本地优先的
- 实用的
- 可理解的
- 负担得起的
- 尊重真实片场压力的

不强制上云。  
不制造不必要的订阅。  
不假装小剧组不存在。  
不把“专业流程”做成只有大团队才配拥有的东西。

## 关于名字

**321Doit** 的意思是：

准备好，检查好，然后开始做。

项目最初从安全拷卡起步，但 **321Doit** 始终是正式产品名称。今天的名字代表的是：准备好、检查好，然后把整条视频工作流真正做下去。
